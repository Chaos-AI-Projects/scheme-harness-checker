# harness-checker

A static analysis tool for Scheme (ChezScheme) programs generated by LLMs. Validates that generated code is safe to execute by checking function whitelists, parameter constraints, and termination properties before running it.

## Features / Analysis Passes

| Pass | Status |
|------|--------|
| Function Whitelist Checking | Implemented |
| Parameter Validation | Planned |
| Termination Analysis | Planned |

### Function Whitelist Checking

Ensures every function call in the program uses only approved procedures. Uses a deny-by-default approach — any identifier not explicitly listed in the whitelist is flagged as a violation.

### Parameter Validation (planned)

Verifies that arguments passed to whitelisted functions satisfy known constraints: correct arity, expected types, and valid value ranges.

### Termination Analysis (planned)

Detects programs that may not terminate (infinite loops / unbounded recursion) using conservative heuristics tuned for typical LLM-generated code patterns.

## Prerequisites

- [Chez Scheme](https://cisco.github.io/ChezScheme/) (tested with ChezScheme 10.x)
- [packrat-extended](https://github.com/ChaosEternal/packrat-extended) — PEG parser library for s-expression pattern matching. Expected at `../packrat-extended` relative to the project root (or provide the path via `--libdirs`).

## Usage

```sh
scheme --libdirs src:<path-to-packrat-extended> --program src/main.ss <source-file> <whitelist-file>
```

**Exit codes:**
- `0` — no violations found (program is safe)
- `1` — violations detected

**Example (clean):**

```
$ scheme --libdirs src --program src/main.ss tests/sample-safe.ss data/minimal-safe.txt
OK: No whitelist violations found.
```

**Example (violations):**

```
$ scheme --libdirs src --program src/main.ss tests/sample-unsafe.ss data/minimal-safe.txt
VIOLATIONS FOUND:
  - system (call in bad-sort body)
  - delete-file (call in cleanup body)
2 violation(s) total.
```

## Whitelist Format

The whitelist is a plain text file with one allowed identifier per line. Lines starting with `;` are comments.

```
; Arithmetic
+
-
*
/

; Boolean
not
boolean?
```

The default whitelist is provided at `data/minimal-safe.txt` and covers safe R6RS/R7RS procedures (arithmetic, list operations, string handling, basic output).

## Running Tests

```sh
cd tests
scheme --libdirs ../src:<path-to-packrat-extended> --program test-whitelist-checker.ss
```

The test suite exercises scoping rules (define, lambda, let, letrec, named let, do, case-lambda, guard, parameterize), forbidden forms (macros), and realistic LLM-generated code patterns (fibonacci, quicksort).

## Scope

- **Target language**: Scheme (ChezScheme dialect)
- **Target input**: LLM-generated programs — expected to be relatively simple and well-structured
- **Out of scope**: Macros (`syntax-case`, `define-syntax`, etc.) are forbidden and flagged as violations
