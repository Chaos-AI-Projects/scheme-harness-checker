# harness-checker

A static analysis tool for Scheme (ChezScheme) programs generated by LLMs. Validates that generated code is safe to execute by performing three major checks before running it.

## Design Overview

### 1. Function Whitelist Checking (#228)

Ensures every function call in the program uses only approved procedures.

- **Whitelist format**: A configuration file listing allowed Scheme procedures (e.g., standard R6RS/R7RS procedures, approved ChezScheme extensions)
- **Analysis**: Parse the Scheme source and extract all call-site identifiers
- **Output**: Report any function calls not present in the whitelist, with source location

### 2. Parameter Validation (#229)

Verifies that arguments passed to whitelisted functions satisfy known constraints.

- **Constraint definitions**: Per-function rules specifying arity, argument types, and valid value ranges
- **Analysis**: At each call site, statically check that arguments match the function's constraint spec
- **Output**: Report violations (wrong arity, type mismatches, out-of-range values) with source location

### 3. Termination Analysis (#230)

Detects programs that may not terminate (infinite loops / unbounded recursion).

- **Loop detection**: Identify `do`, named `let`, and other looping constructs; verify exit conditions exist
- **Recursion analysis**: For recursive functions, check for decreasing arguments and reachable base cases
- **Bounded approach**: Since the halting problem is undecidable in general, use conservative heuristics tuned for typical LLM-generated code patterns. Flag programs that cannot be proven to terminate within defined bounds.

## Scope

- **Target language**: Scheme (ChezScheme dialect)
- **Target input**: LLM-generated programs — expected to be relatively simple and well-structured
- **Out of scope**: Macros (`syntax-case`, `define-syntax`, etc.) are forbidden
